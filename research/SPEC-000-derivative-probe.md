# SPEC-000: The Derivative Probe (gating experiment #0)

> This is the first experiment. It gates everything downstream. Do not build
> grip mechanisms until this has a result.

## The question

Is the certainty-**trajectory** of a model's belief already present in the
hidden state of a vanilla transformer, or is it **absent**?

This tests the Portfolio Mind amnesia claim directly: that current models are
"architecturally amnesiac about their own certainty trajectories." If the
*trajectory* (slope, acceleration of belief) is present in `h[t]`, the model
"knows but isn't using it," and the minimal grip intervention is a **wire**
(a selector head that routes existing info). If it's absent, grip state must
be **added**. Either result is publishable; the result bifurcates architecture.

## Why level vs. derivative matters

- **Level targets** (posterior `p[h|e_{<=t}]`, entropy at `t`): will probe
  **well**, because the model must compute them to predict. A positive result
  is *uninformative*. This is the control.
- **Derivative targets** (`d_conf[t] = topmass[t] - topmass[t-1]`,
  `dd_conf[t] = topmass[t] - 2*topmass[t-1] + topmass[t-2]`): representing these
  requires the model to have **retained** its t-1 / t-2 confidence in a readable
  form. This is the actual test.

## Data: Bayesian Evidence Streams (exact spec)

A stream is generated by forward-simulating Bayesian belief update. All latents
are therefore known exactly.

### Generative model

- `K` hypotheses, one true `h* ~ Uniform(K)`.
- `S` sources, each with hidden reliability `r_s ~ Beta(a,b)` drawn per stream.
- At each step `t`, draw a source `s_t`, then emit an evidence token. With
  probability `r_{s_t}` the token is **informative**: drawn from a hypothesis-
  conditional distribution `P(token | h*)`. With probability `1-r_{s_t}` it is
  a **decoy** drawn from the marginal `P(token)`.
- **Late reliability reversal:** with fixed probability, after some step `t_rev`
  a source's reliability flips (high->low or low->high) without an explicit
  token marking it. The model must infer the new trust from behavior.
- Vocab `<= 64`, human-readable symbol classes (e.g. `H0_SAY_A`, `DECOY`, ...).

### Belief update (ground truth)

Maintain a posterior `p[t] in simplex^K`, prior uniform. At each step update via
Bayes rule with the *true* generative probabilities (so the posterior is the
exact Bayesian-optimal belief given the stream). From `p[t]` derive:
- `topmass[t] = max_k p[t,k]`
- `entropy[t] = -sum p[t,k] log p[t,k]`
- `d_conf[t]`, `dd_conf[t]` as above (zero-padded at the boundaries).
- `decisive_idx[t] = 1` iff `|d_conf[t]|` is in the top 5% of the stream.

### Non-negotiables

1. **Deterministic given seed.** One seed => one exact stream.
2. **Fixed `T` per sample, padded.** The generator must NEVER emit variable
   length (pytorch #181213 MPS leak). Padding token is a reserved id.
3. Emit `d_conf`, `dd_conf`, `source_trust`, `decisive_idx` as first-class
   array fields on `StreamSample`, not derived after the fact.
4. `block_boundaries` partitions `[0,T)` into `ceil(T/block_size)` contiguous
   blocks. Used by sparse attention and the decisive-token-recall metric.

## Experiment procedure

1. Train a tiny **dense** transformer (`DenseTransformer`, ~1-4M params,
   `d_model=128`, `n_layers=4`, seq 512) on next-token prediction + an auxiliary
   posterior head, on a large pool of generated streams. Train to convergence
   on a small set (overfit-OK for the probe — we are asking about
   *representational capacity*, not generalization).
2. **Freeze** the backbone. Collect hidden states `h[t]` over a held-out set of
   streams along with their ground-truth latents.
3. Fit a **linear probe** (`nn.Linear(d_model, 1)`) from `h[t]` to each target:
   `topmass`, `entropy` (level controls), `d_conf`, `dd_conf` (derivatives),
   and `source_trust[t]` (routing). Train on 80% of (stream, t) positions,
   eval R^2 + MSE on the held-out 20%.
4. Report a table: target x {R^2, MSE}. At least 3 seeds.

## Decision rule (pre-registered)

- **Level probes succeed, derivative probes fail (R^2 near 0):** amnesia holds.
  grip state must be ADDED. Proceed to full grip mechanism design.
- **Derivative probes also succeed (R^2 clearly above a noise baseline):**
  trajectory info is present but not routed to selection. Minimal grip = a wire
  (selector head on frozen backbone). Cheaper, sharper result.
- **Partial / mixed:** report which derivatives are present. `d_conf` present
  but `dd_conf` absent is itself informative about *how much* trajectory is kept.

## Leakage guard

The probe target is `d_conf` (a *change* in belief), not the belief itself.
Even a perfect `d_conf` probe cannot decode the answer from a single `h[t]`,
because `d_conf` is a difference and the level must be recovered separately.
This is the principle made architectural: grip reconstructs *meta*-variables,
not the label. Verify by also probing `h[t] -> answer`: if that probes well,
the backbone is leaking and the task needs hardening.

## Acceptance

- `pytest` green.
- Stream generator produces human-readable samples; posterior sanity-checked
  against a hand-computed 2-step example.
- Probe runs end-to-end; the R^2 table is written to `runs/probe-000/report.json`.
